# 2026-06-13

## 铁路RAG项目 — 混合检索实现与评估

### 完成内容
1. 实现了混合检索 `HybridRetriever`（FAISS向量 + BM25字符级n-gram关键词，RRF融合），代码在 `retriever.py`
2. 实现了问题改写 `QueryRewriter`，但测试发现改写反而降低表现，已从评估中移除
3. 运行了 3 模式评估（纯LLM / 向量RAG / 混合RAG），5题结果

### 关键发现
- **Simple 题 RAG 反降原因**：RAG Prompt 的"严格依据参考信息"约束使模型生成过度简洁的回答，而纯LLM展开解释更贴近 ground truth
- **混合检索优势**：Q2 忘带身份证问题，向量RAG=0.1589 → 混合RAG=0.7195（+0.5606），关键词匹配补救了纯语义检索的盲区
- **混合检索劣势**：Q3 证件列表题，两者都失败（0.0165），说明检索本身未命中正确的chunk

- `query_rewriter.py`: 问题改写模块（暂时闲置）
